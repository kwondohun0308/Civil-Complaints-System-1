# 프로젝트 회고문 (RETROSPECTIVE)

> **프로젝트**: AI-Civil-Affairs-Systems — 온디바이스 LLM 기반 민원 검색·구조화·질의응답 시스템  
> **기간**: 2026년 1학기  
> **팀 구성**: BE1(팀장/데이터 파이프라인), FE(Next.js UI), BE2(검색/벡터DB), BE3(API/LLM/RAG)  
> **작성일**: 2026-06-19

---

## 1. 프로젝트 개요

민원 담당자를 위한 온디바이스(로컬 실행) AI 시스템을 구축하는 것이 목표였다.
외부 클라우드 API에 의존하지 않고, Ollama 기반의 로컬 LLM과 ChromaDB 벡터 검색을 결합하여 민원 데이터를 구조화하고, 근거 기반 답변 초안을 생성하는 파이프라인을 완성했다.

## 2. 잘한 점 (Keep)

### 2.1 온디바이스 아키텍처 선택
- 민원 데이터의 개인정보 보호 요구사항을 외부 API 호출 없이 충족할 수 있었다.
- Ollama + EXAONE 3.5 7.8B 모델 조합으로 한국어 민원 도메인에서 실용적인 성능을 확인했다.

### 2.2 모듈화된 파이프라인 설계
- Ingestion → Structuring → Retrieval → Generation 각 단계가 독립 모듈로 분리되어 있어, 개별 단계의 실험과 교체가 용이했다.
- Adaptive Router를 통한 검색 전략 자동 선택(topic/complexity 기반)이 검색 품질 향상에 기여했다.

### 2.3 풍부한 테스트 커버리지
- 63개의 단위 테스트 파일을 통해 핵심 로직(구조화, 검색 라우팅, 생성 파이프라인, 안전 규칙 등)을 검증했다.
- 오프라인 실행 가능한 순수 단위 테스트와 인프라 의존 통합 테스트를 분리하여 CI 파이프라인에서도 안정적으로 실행할 수 있도록 설계했다.

### 2.4 Hybrid 검색 전략 채택
- Dense(BGE-M3) + BM25 + RRF 앙상블 검색이 전 지표에서 1위를 기록하여 기본 전략으로 채택했다.
- 검색 평가 프레임워크를 통해 정량적 비교 실험을 수행하고 결과를 문서화했다.

### 2.5 프론트엔드-백엔드 분리
- Next.js Workbench UI와 FastAPI 백엔드를 완전히 분리하여 각 팀원이 독립적으로 개발할 수 있었다.

## 3. 아쉬운 점 (Problem)

### 3.1 초기 CI/CD 부재
- 프로젝트 초반에는 GitHub Actions CI가 구성되지 않아 PR 머지 시 린트/테스트 게이트 없이 코드가 통합되었다.
- 프로젝트 후반부에 CI 파이프라인(린트 + 단위 테스트), Dependabot 보안 스캔, DORA 메트릭 수집을 추가하여 보완했다.

### 3.2 LLM 의존 테스트의 불안정성
- Ollama 서버 상태, 모델 로딩 시간, GPU 가용성에 따라 통합 테스트 결과가 비결정적이었다.
- 향후에는 LLM 응답 mock/fixture를 더 체계적으로 관리할 필요가 있다.

### 3.3 배포 자동화 미비
- 로컬 온디바이스 프로젝트 특성상 클라우드 배포 파이프라인이 불필요했지만, 로컬 환경 재현성(Docker Compose 등)을 더 일찍 준비했으면 팀원 간 환경 차이 문제를 줄일 수 있었을 것이다.

### 3.4 문서화 시점
- ADR(Architecture Decision Records)이나 Runbook 같은 운영 문서가 프로젝트 후반에 작성되었다.
- 의사결정 기록을 개발 진행과 동시에 남겼으면 코드 리뷰와 온보딩이 더 수월했을 것이다.

## 4. 시도할 것 (Try)

### 4.1 Docker Compose 기반 원클릭 실행 환경
- Ollama + FastAPI + Next.js + ChromaDB를 하나의 `docker-compose.yml`로 묶어 `docker compose up` 한 번으로 전체 스택을 구동할 수 있도록 개선한다.

### 4.2 LLM 응답 스냅샷 테스트
- 주요 프롬프트에 대한 LLM 응답을 스냅샷으로 저장하고, 모델 업그레이드 시 회귀를 탐지하는 메커니즘을 도입한다.

### 4.3 관측성 강화
- Prometheus 메트릭 엔드포인트(`/metrics`)와 Grafana 대시보드 템플릿을 추가하여 API 응답 시간, 에러율, 검색 지연 시간을 실시간으로 모니터링한다.

### 4.4 E2E 평가 자동화
- mini-eval(소규모 골든셋 기반 검색+생성 품질 평가)을 CI 파이프라인에 통합하여 모든 PR에서 품질 회귀를 자동 탐지한다.

## 5. 핵심 교훈

| 영역 | 교훈 |
|------|------|
| 아키텍처 | 온디바이스 LLM은 보안/프라이버시 측면에서 강점이 크지만, 하드웨어 의존성 관리가 핵심 과제다 |
| 검색 | Hybrid(Dense+BM25+RRF)가 단일 전략 대비 일관된 성능 우위를 보여주며, 적응형 라우팅이 도메인별 최적화에 효과적이다 |
| 테스트 | 외부 의존성(LLM, 벡터DB)이 있는 시스템에서는 테스트 계층을 명확히 분리하는 것이 필수적이다 |
| CI/CD | 프로젝트 초기에 CI 게이트를 설정하면 후반부의 통합 비용을 크게 절감할 수 있다 |
| 협업 | Main-Feature 브랜치 전략과 Discord 알림 연동이 4인 팀의 작업 동기화에 효과적이었다 |
