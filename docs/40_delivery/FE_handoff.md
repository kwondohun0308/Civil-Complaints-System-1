# FE 핸드오프 — 구조화·긴급도·법령 인용 화면 렌더링 가이드

BE1 구조화 + 긴급도 + BE3 답변에 들어오는 새 필드를 **어떻게 화면에 표시하면 되는지** 정리합니다.
핵심 원칙: 이 신호들은 대부분 **"후보(soft)"** 라 confidence를 절대수치로 강조하지 말고 *배지·칩·상대 강도*로 보여주세요.

---

## 1. 한눈에 — 어떤 필드를 어떻게

| 필드 (출처) | 화면 요소 | 비고 |
| --- | --- | --- |
| `urgency.level` (구조화) | **긴급도 배지** | 긴급=빨강, 높음=주황, 보통=노랑, 낮음=회색 |
| `urgency.factors`/`evidence`/`override` | 배지 **툴팁** | "안전 위협 감지(가스 냄새)" 등 근거 |
| `legal_citations[].public_url` (답변) | **조문 링크** | `public_url`만 사용(아래 ⚠️) |
| `legal_citation_warnings` (답변) | **경고 배너** | "초안에서 미검증 법령 인용이 제거됨" |
| `responsible_unit[]` (구조화) | **담당부서 후보 칩** | name + 신뢰도 표시, "후보" 라벨 |
| `legal_refs[]` (구조화) | **관련 법령 칩** | name(+조문 링크 가능), "관련 후보" |
| `entity_texts[]` (구조화) | **대상 객체 태그** | 지게차, 가로등 등 |
| `key_terms[]` (구조화) | **키워드 태그** | 중요도 순 |
| `title` (목록/케이스) | **민원 제목** | 서버에서 정제되어 전달(아래) |

---

## 1-1. 제목(`title`) — 서버에서 정제되어 옴 (그대로 표시)

케이스 목록·검색 결과의 `title`은 공용 함수(`app/core/title_builder.build_case_title`)를 거쳐 전달되며 retrieval/UI 어댑터가 동일 규칙을 씁니다. **FE는 별도 가공 없이 그대로 표시**하면 됩니다.

- **'고객:' 제거**: 상담 원문 화자 라벨(선행 `고객:`/`고객 :`)이 제거된 상태로 옴. (본문 중간 "고객센터" 등은 보존)
- **빈 title 자동 보정**: 제목 소스가 모두 비면 `"{카테고리} 관련 민원"`으로 채워져 옴 → **`title`은 항상 비어있지 않음**(FE에서 빈 제목 폴백 불필요).
- 길이는 60자 기준으로 잘리고 말줄임(`...`) 처리되어 옴.

## 2. 긴급도 배지 (가장 눈에 띄게)

```jsonc
"urgency": {
  "level": "긴급",                  // 낮음|보통|높음|긴급
  "score": 0.86,
  "factors": {"safety": 2, "ongoing": true, "recurring": true, "explicit_deadline": false},
  "evidence": ["가스 냄새", "폭발"],
  "override": "safety"              // "safety"면 생명위협으로 긴급 승격
}
```
- **색상**: 긴급 🔴 / 높음 🟠 / 보통 🟡 / 낮음 ⚪.
- **툴팁/상세**: `evidence`(근거 문구) + `factors`(안전/반복/기한). `override === "safety"`면 "⚠ 생명·안전 위협" 강조.
- 목록 정렬·필터에 `level` 사용(긴급 우선 노출). `score`는 내부 정렬 보조(미보정이라 % 강조 X).
- (참고) 기존 `priority`(매우급함/급함/보통)도 있으나, **신규 `urgency`를 우선** 사용 권장.

## 3. 법령 조문 인용 (답변 화면)

```jsonc
"legal_citations": [
  {"law_name": "건축법", "article_no": "제80조", "law_id": "001823",
   "public_url": "https://www.law.go.kr/법령/건축법/제80조",
   "verified": true}
],
"legal_citation_warnings": ["미검증 인용 제거: 건축법 제999조"]
```
- 답변 본문의 "건축법 제80조" 옆/아래에 **`public_url` 링크**(새 탭). 라벨: `건축법 제80조 ↗`.
- 공개 API에는 `source_url`이 포함되지 않는다. 법령 링크는 반드시 **`public_url`**만 사용한다.
- `legal_citation_warnings`가 있으면 답변 하단에 작은 경고 배너: *"초안에 근거가 확인되지 않은 법령 인용이 있어 자동 제거되었습니다."*
- 답변 본문에 `[미검증 인용 제거]` 마커가 보이면 그대로 두거나 회색 처리(검토자에게 제거 흔적 표시).

## 4. 후보 칩들 (담당부서·법령·객체)

```jsonc
"responsible_unit": [{"name": "건설기계과", "confidence": 0.78, "evidence": ["지게차","건설기계"]}],
"legal_refs":       [{"name": "건설기계관리법", "confidence": 0.6, "law_id": "000239", "evidence": ["지게차"]}],
"entity_texts":     [{"text": "지게차", "label": "OBJECT", "confidence": 0.9, "evidence": ["미만 지게차"]}],
"key_terms":        ["지게차", "적성검사", "면허", "갱신"]
```
- **칩(chip/tag)** 형태로 표시. 호버 시 `evidence`(근거 문구) 툴팁.
- **신뢰도 표기**: 숫자(0.78)를 그대로 크게 쓰지 말고 **상대 등급**으로 — 예: `≥0.8 높음(진한색)`, `0.5~0.8 보통`, `<0.5 낮음(연한색/회색)`. 또는 칩 옆 점 3개(●●○).
- `responsible_unit`·`legal_refs`는 **"후보"** 임을 라벨로 명시(예: "담당부서(후보)"). 빈 배열일 수 있음(아래).
- `legal_refs` 칩 클릭 → 해당 법령 공개 페이지(`https://www.law.go.kr/법령/{name}`)로 링크 가능.

---

## 5. 빈 값/없을 때 처리 (중요)

- `responsible_unit: []` 가 자주 옵니다 — 임베딩 인덱스가 꺼져 있으면 비어 옵니다(서버 설정). 빈 배열이면 **섹션 숨김**.
- `legal_citations`와 `legal_citation_warnings`는 항상 배열로 오며, 비어 있으면 해당 영역을 숨긴다.
- 어떤 후보 배열이든 비어 있으면 그 칩 영역을 그리지 않습니다(빈 상태 텍스트 불필요).

---

## 6. UX 원칙 (정직)

- 이 신호들은 **검색·분류 보조 후보**라 100% 정답이 아닙니다. confidence는 미보정 → **단정적 % 노출 금지**, 배지/칩/상대 강도로.
- 긴급도·법령 인용은 **검토 보조**입니다 — 화면 어딘가에 "AI 보조 분석, 최종 판단은 담당자" 류 고지를 권장.
- 색맹 대비: 긴급도 배지는 색 + 텍스트(긴급/높음/…) 같이 표기.
