# LLM-Rubric 평가 근거 및 Baseline 검토

## 1. 문서 목적

현재 민원 회신 평가기인
`scripts/evaluate_llm_rubric_civil_replies.py`의 평가 항목, 점수 가중치,
개별 점수 산정 방식과 비교 baseline을 검토한다.

주요 참고 논문은 다음과 같다.

- LLM-Rubric: A Multidimensional, Calibrated Approach to Automated Evaluation of Natural Language Texts
- 논문 원문: <https://arxiv.org/html/2501.00274v1>

검토 기준일은 2026년 6월 12일이다. 현재 기본 평가기 버전은
`llm_rubric_proxy_civil_replies_v3_generated_body`다.

## 2. 핵심 결론

현재 구현은 논문의 Q0~Q8 평가 관점을 참고했지만, 논문의 평가기를
그대로 재현한 것은 아니다.

- Q1, Q3~Q7은 논문의 평가 차원과 대체로 직접 대응한다.
- Q2는 논문에서 "주어진 문서만으로 질문을 해결할 수 있는가"를 묻지만,
  현재 구현은 citation, 답변 구체성, 참조 답안 정렬도를 함께 평가한다.
- Q8은 논문에서 대화 turn 수의 적절성을 평가하지만, 현재 구현은 단일 민원
  회신의 업무 완결성을 평가하도록 재정의했다.
- 논문은 Q0~Q8에 고정 가중치를 제시하지 않는다. 사람 평가를 목표값으로 삼아
  calibration network가 결합 방식을 학습한다.
- 현재 가중치와 0~10점 산식, 품질 상한은 프로젝트가 자체 설계한
  deterministic proxy다.
- 기본 평가 범위는 `generated_body`다. 고정 회신 틀인 1·2·4문단은
  Q0·Q1·Q2·Q5·Q6·Q7·Q8의 텍스트 평가에서 제외하고, 모델이 민원별로
  작성한 3문단의 실질 본문만 평가한다.
- Q3·Q4는 답변 문자열이 아니라 구조화 citation 필드를 평가하므로
  `generated_body` 전환 전후 산식이 동일하다.
- 고정 회신 틀의 1~4문단 존재 여부와 종료 문구 중복은 점수에 포함하지 않고
  `reply_shell_diagnostics`로 별도 보고한다.

따라서 현재 평가기는 다음과 같이 표현하는 것이 정확하다.

> LLM-Rubric의 다차원 평가 관점을 민원 회신 업무에 적용한 규칙 기반
> deterministic proxy

## 3. 평가 항목 선정 근거

논문은 자연어 생성 평가와 RAG 평가 문헌을 바탕으로 전체 만족도와 함께
자연스러움, 근거 충분성, 인용 품질, 중복성, 간결성, 대화 효율성을 평가한다.
각 항목이 전체 만족도 예측에 제공하는 정보를 검증하기 위해 항목 제거
ablation도 수행한다.

| ID | 현재 평가 항목 | 논문의 원래 의미 | 대응 정도 | 검토 의견 |
| --- | --- | --- | --- | --- |
| Q0 | 종합 만족도 | 사용자의 전체적인 만족 가능성 | 직접 대응 | 현재는 사람 만족도 예측이 아니라 Q1~Q8의 규칙 기반 합산값이다. |
| Q1 | 생성 본문 품질 | 자연스러운 어조와 대화 품질 | 직접 대응 | 고정 회신 틀을 제외하고 공공기관 문체, 내부 라벨, 이스케이프·디버그 노출을 평가한다. |
| Q2 | 근거 충분성 | 제공된 문서가 질문 해결에 충분한가 | 부분 대응 | 현재는 답변 구체성까지 포함하므로 retrieval adequacy와 answer groundedness가 혼합된다. |
| Q3 | 인용 포함 | 주요 주장에 인용이 포함되는가 | 직접 대응 | 구조화 citation 수와 매칭률을 사용하는 현재 방식과 잘 대응한다. |
| Q4 | 인용 정확성 | 인용이 주장을 실제로 뒷받침하는가 | 직접 대응 | `citation_support_rate_strict` 사용이 논문의 취지와 가장 가깝다. |
| Q5 | 최적 출처성 | 제공된 문서 중 최선의 출처를 사용했는가 | 직접 대응 | strict citation과 참조 답안 앵커를 함께 사용한다. |
| Q6 | 중복 없음 | 반복 또는 불필요한 정보가 없는가 | 직접 대응 | 생성 본문에서 템플릿, 구조 문자열, 내부 라벨, 디버그 노출을 평가한다. |
| Q7 | 길이·밀도 | 답변이 적절하게 간결한가 | 직접 대응 | 고정 문단을 제거한 생성 본문과 `consultant_answer` 실질 본문의 길이·문장 수 분포를 비교한다. |
| Q8 | 업무 완결성 | 필요한 대화 turn 수가 적절한가 | 재정의 | 생성 본문 안의 민원 요지, 검토 판단, 조치와 제약을 평가하며 고정 후속 안내는 점수에서 제외한다. |

### 3.1 프로젝트 관점의 합리적 이유

민원 회신은 일반 대화와 달리 다음 조건을 동시에 만족해야 한다.

1. 공공기관 회신 형식과 문체를 지켜야 한다.
2. 법령, 검색 문서 또는 실제 행정 근거에 기반해야 한다.
3. 인용이 존재하는 것뿐 아니라 인용 내용이 실제 주장을 지지해야 한다.
4. 소관과 권한을 넘는 확정 약속을 하지 않아야 한다.
5. 민원 요지, 판단, 제약, 조치와 후속 안내가 한 번의 회신에서 완료되어야 한다.

이 때문에 논문의 평가 차원을 그대로 복사하기보다 Q1, Q2, Q6, Q7, Q8을
민원 회신에 맞게 변형한 것은 타당하다. 다만 변형된 항목은 논문 자체가
직접 검증한 항목이라고 주장해서는 안 된다.

공통 회신 형식의 준수 여부와 민원별 생성 내용의 품질은 서로 다른 문제다.
따라서 현재 구현은 전자를 `reply_shell_diagnostics`, 후자를 Q0~Q8 점수로
분리한다. 이 분리는 모든 답변에 동일하게 삽입된 문장이 Q1·Q6을 자동으로
높이는 것을 방지한다.

## 4. 현재 가중치와 근거 수준

현재 코드의 Q0 가중치는 다음과 같다.

| 항목 | 현재 가중치 |
| --- | ---: |
| Q1 회신 품질 | 0.14 |
| Q2 근거 충분성 | 0.14 |
| Q3 인용 포함 | 0.10 |
| Q4 인용 정확성 | 0.14 |
| Q5 최적 출처성 | 0.08 |
| Q6 중복 없음 | 0.12 |
| Q7 길이·밀도 | 0.10 |
| Q8 업무 완결성 | 0.18 |
| 합계 | 1.00 |

이 값은 2026년 6월 11일 커밋 `0d53331`에서 현재 형태로 도입됐다.
저장소 문서와 커밋 이력에서는 각 값을 결정한 사람 평가, 통계 분석 또는
논문 기반 계산 과정을 찾지 못했다.

따라서 현재 가중치는 다음 수준의 근거를 갖는다.

- 논문 직접 근거: 없음
- 프로젝트 업무 중요도에 따른 합리적 휴리스틱: 있음
- 민원 사람 평가 데이터에 의한 실증 검증: 없음
- 사람 만족도와의 상관관계 또는 예측 오차 검증: 없음

현재 가중치는 임시 운영값으로는 사용할 수 있지만, 논문 기반 최적 가중치라고
표현해서는 안 된다.

## 5. 논문 Ablation을 이용한 참고 가중치

논문의 real-data ablation에서 각 항목을 제거했을 때 증가한 RMSE를
Q1~Q8 범위에서 단순 정규화하면 다음 참고값을 얻을 수 있다.

| 항목 | 현재 가중치 | Ablation 참고값 |
| --- | ---: | ---: |
| Q1 | 14.0% | 6.8% |
| Q2 | 14.0% | 8.2% |
| Q3 | 10.0% | 25.2% |
| Q4 | 14.0% | 12.5% |
| Q5 | 8.0% | 14.0% |
| Q6 | 12.0% | 0.3% |
| Q7 | 10.0% | 18.3% |
| Q8 | 18.0% | 14.7% |

이 표는 후보 가중치를 만들기 위한 참고 자료일 뿐이다.

- 논문의 데이터는 현재 한국어 민원 데이터와 다르다.
- 현재 Q2와 Q8은 논문의 질문과 의미가 다르다.
- ablation의 RMSE 증가량은 학습된 모델 내부의 기여도이며, 고정 선형
  가중치와 동일하지 않다.

다만 현재 설정에서 Q6가 상대적으로 높고 Q3, Q5, Q7이 낮을 가능성을
점검해야 한다는 실험 가설은 제공한다.

## 6. 현재 개별 점수 평가 근거

| 항목 | 현재 주요 입력 | 현재 산정 개요 |
| --- | --- | --- |
| Q1 | 생성 본문의 문체, 내부 라벨, 이스케이프, 템플릿, 디버그 노출 | 10점에서 실제 생성 문장의 부자연스러움과 출력 노이즈를 감점한다. 고정 인사·마무리는 평가하지 않는다. |
| Q2 | strict citation, citation 수, 구체성, 참조 정렬도 | strict match 비중을 가장 크게 두고 구체성과 참조 정렬을 가산한다. |
| Q3 | strict/repaired citation 수와 매칭률 | strict citation을 우대하며 후처리 citation만 있으면 6점으로 제한한다. |
| Q4 | citation support 또는 identity match | snippet 지지율이 있으면 이를 10점으로 환산하고, ID 일치만 있으면 최대 7점으로 제한한다. |
| Q5 | strict 출처, 참조 정렬, 핵심 앵커 | 실제 회신과 핵심 내용이 정렬되는지 평가하고 의미 역전 시 감점한다. |
| Q6 | 생성 본문의 반복률, 템플릿, 구조 문자열, 내부 라벨, 디버그 정보 | 10점에서 반복과 출력 노이즈를 감점한다. 고정 종료 문구 중복은 별도 shell 진단으로 분리한다. |
| Q7 | 생성 본문 길이, 문장 수, 1:1 참조 본문 길이 | 실제 회신 9,130건에서 공통 인사·문의 안내를 제거한 실질 본문의 분위수와 비교한다. |
| Q8 | 생성 본문의 요지, 검토, 조치, 제약, 구체성 | 고정 후속 안내를 제외하고 업무 판단에 필요한 구성요소를 가산하며 일반론과 의미 오류를 감점한다. |

현재 점수는 코드로 재현 가능하고 케이스별 감점 사유를 출력한다는 장점이 있다.
그러나 일부 임계값과 배점도 사람 평가로 보정된 값이 아니므로 가중치와
동일하게 휴리스틱으로 분류해야 한다.

### 6.1 평가 범위와 모범 답안 정규화

`generated_body` 모드에서는 생성 답변과 모범 답안을 같은 범위로 비교한다.

- 생성 답변: `3. 검토 의견은 다음과 같습니다.` 이후부터 4문단 시작 전까지
- 모범 답안: `consultant_answer`에서 인사말, 번호 접두사, 일반적인 문의 안내
  등 공통 회신 표현을 제거한 실질 본문
- 추출 실패: 고정 회신 틀이 없는 기존 답변과의 호환을 위해 전체 답변을
  생성 본문으로 간주
- 빈 답변: 생성 본문 길이 0으로 처리하고 Q0~Q8을 0점으로 평가

과거 결과 재현이 필요한 경우에만 `--evaluation-scope full_reply`를 사용한다.
두 평가 범위의 점수는 동일한 지표명이라도 직접적인 시계열 비교 대상으로
사용해서는 안 된다.

## 7. Q0 품질 상한 검토

Q0는 단순 가중 평균에 다음 품질 상한을 추가 적용한다.

| 조건 | Q0 최대점 |
| --- | ---: |
| 답변 없음 | 0.0 |
| 검색 메타데이터 또는 디버그 노출 | 3.0 |
| 구조 문자열 노출 또는 citation 없음 | 4.0 |
| 실제 회신의 조건부·불가 결론을 확정 조치로 역전 | 3.5 |
| 관리 주체가 다른 사안에 행정기관 직접 조치를 약속 | 4.0 |
| paired reference와 정렬이 전혀 없음 | 4.5 |
| 법령 grounding 오류 또는 매우 낮은 참조 정렬 | 5.0 |
| 근거 없는 강한 이행 약속 | 5.0 |
| 확인되지 않은 현재 상태·일정·완료 사실을 단정 | 5.5 |
| 일반 템플릿 남용 | 5.5 |
| strict citation 없이 후처리 citation만 존재 | 6.5 |

이 상한은 치명적인 행정 오류가 높은 부분 점수에 가려지는 것을 방지한다는
점에서 합리적이다. 반면 상한 적용 여부에 따라 Q0 분포가 크게 달라질 수 있다.
따라서 평균 점수와 함께 다음 통계를 반드시 보고해야 한다.

- 상한 사유별 적용 건수와 비율
- 상한 적용 전후 Q0 평균
- 상한 때문에 순위가 변경된 케이스 수
- 각 상한 조건의 사람 평가 precision과 recall

동일한 EXAONE 50건 결과를 기존 전체 회신 평가와 `generated_body` 평가로
비교한 결과는 다음과 같다.

| 항목 | 기존 전체 회신 | 생성 본문 전용 | 변화 |

| Q0 | 5.944 | 5.738 | -0.206 |
| Q1 | 9.390 | 7.930 | -1.460 |
| Q2 | 7.686 | 7.616 | -0.070 |

Q1·Q6·Q7의 하락은 고정 문단이 제공하던 자연스러운 문체, 종료 표현과

형식 보너스를 제거한 의도된 변화임을 보여준다.

### 7.2 생성 안전성 보강

2026년 6월 12일 수동 검토에서 결론 반전, 소관 권한 초과, 유사 사례 사실
혼입, 확인되지 않은 일정 단정이 규칙 점수보다 자주 발견됐다. 이에 평가기와
생성 경로를 다음과 같이 보강했다.

- `disposition_reversal`: 실제 회신의 불가·조건부 결론을 승인·실행 약속으로
  뒤집은 경우
- `authority_mismatch`: 사유지·소유자·관리사무소 소관 사안에 행정기관의
  직접 조치를 약속한 경우
- `unsupported_commitment`: 모범 답안에 없는 설치·철거·이동·신설·건설·
  매입·개방·지정·예산 확보 등을 확약한 경우
- `unverified_current_fact`: `확인하였습니다`, `보고되었습니다`,
  `진행 중입니다`, `완료되었습니다`처럼 현재 상태를 근거 없이 단정한 경우
- `CONTEXT_CONSTRAINT_CONFLICT`: 검색 근거의 처리 불가·소관 제약과 생성
  답변의 확정 조치가 충돌하는 운영 경고

평가기 semantic risk는 paired `consultant_answer`가 있을 때만 결론 정합성을
판정한다. 운영 API의 경고는 검색 근거를 사용하므로 정답 판정이 아니라
검토 필요 신호이며, 단독으로 답변을 자동 승인하거나 반려하는 용도로 쓰지
않는다.

## 8. 권장 비교 Baseline

### 8.1 평가 방식 Baseline

동일한 생성 답변과 사람 평가셋에서 다음 방식을 비교한다.

모든 baseline은 같은 `evaluation_scope`, 같은 answer field, 같은 reference
snapshot을 사용해야 한다. 특히 `full_reply` 결과와 `generated_body` 결과를
모델 성능 차이로 해석하면 안 된다.

| ID | 평가 방식 | 목적 |
| --- | --- | --- |
| B0 | Random 또는 평균값 예측 | 평가기가 최소 기준보다 의미 있는지 확인 |
| B1 | Direct LLM Judge Q0 | 세부 rubric 없이 전체 만족도만 직접 평가 |
| B2 | Equal Weight | Q1~Q8을 각각 12.5%로 합산 |
| B3 | Current Heuristic | 현재 가중치와 Q0 상한 사용 |
| B4 | Ablation-Inspired Weight | 논문 ablation 참고값을 민원용으로 조정 |
| B5 | Human-Calibrated Linear Model | Q1~Q8과 citation 지표로 Ridge 또는 Logistic Regression 학습 |
| B6 | Learned Calibration Network | 논문과 유사하게 사람 Q0를 예측하도록 작은 신경망 학습 |

최소 비교 세트는 B1, B2, B3, B5다.

### 8.2 생성 시스템 Baseline

평가기뿐 아니라 생성 시스템 성능 비교에서는 다음 답변군을 동일한 평가셋에
포함하는 것이 좋다.

- 실제 `consultant_answer`: 사람 작성 기준선
- 검색 없이 생성한 모델 답변
- Dense 또는 기존 단일 RAG 답변
- 현재 법령 grounding 및 strict citation 적용 답변
- 현재 답변에서 citation 후처리만 제거한 ablation

실제 `consultant_answer`는 절대적인 정답이라기보다 사람 작성 응답의 분포와
도달 가능한 품질 수준을 보여주는 기준선으로 사용한다.

## 9. 권장 검증 지표

사람 평가자는 동일한 0~10 Q0와 필요한 세부 항목을 독립적으로 평가한다.
가능하면 케이스당 2명 이상이 평가하고 불일치는 합의 또는 제3 평가자로
조정한다.

평가기 비교에는 다음 지표를 사용한다.

- MAE: 사람 Q0와 평가기 점수의 평균 절대 오차
- RMSE: 큰 오차에 더 큰 패널티를 주는 예측 오차
- Spearman correlation: 답변 품질 순위 일치도
- Pearson correlation: 점수의 선형 상관관계
- Cohen's kappa 또는 weighted kappa: 점수 구간의 평가자 일치도
- 치명적 오류 탐지 precision, recall, F1
- 상한 적용 전후 calibration error

현재 50건은 초기 진단에는 사용할 수 있지만 가중치 학습과 안정적인
일반화를 위해서는 카테고리별로 분산된 100~300건 이상의 사람 평가가
권장된다.

## 10. 권장 적용 순서

1. 현재 가중치와 상한을 동결하고 버전을 기록한다.
2. 50건에 대해 최소 2명의 사람 Q0 및 세부 항목 평가를 확보한다.
3. Equal Weight, Current Heuristic, Direct LLM Judge를 먼저 비교한다.
4. 상한 사유별 precision과 recall을 측정해 잘못된 상한을 수정한다.
5. 데이터가 100건 이상이면 선형 calibration model을 학습한다.
6. holdout 평가에서 현재 방식보다 MAE, RMSE와 순위 상관이 개선될 때만
   새 가중치를 채택한다.

데이터가 충분하지 않은 현재 단계에서는 가중치를 논문 ablation 값으로 즉시
교체하기보다, 현재값과 Equal Weight를 함께 baseline으로 보존하는 편이
안전하다.

## 12. 최종 판단

현재 평가 항목은 논문의 다차원 평가 관점과 민원 업무 요구를 결합했다는
점에서 진단 도구로 사용할 근거가 있다. 하지만 현재 가중치, 임계값과 Q0
상한은 논문이 제공한 값이 아니며 사람 평가로 보정되지 않았다.

발표와 보고서에서는 다음 네 문장을 명시하는 것이 적절하다.

1. 평가 항목은 LLM-Rubric의 Q0~Q8 관점을 참고했다.
2. 민원 단일 회신에 맞게 일부 항목을 재정의하고 실제 상담 답변 분포로
   규칙 기반 점수를 계산했다.
3. 고정 회신 틀은 별도 형식 진단으로 분리하고, 기본 점수는 민원별 생성
   본문과 구조화 citation을 평가한다.
4. 현재 가중치는 초기 휴리스틱이며, 추후 사람 평가 데이터로 calibration할
   예정이다.
