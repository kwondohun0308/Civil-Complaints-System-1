# Workbench E2E 시연 동선 (Week 7)

## 목표
단일 민원 선택 후 좌/중/우 3단 패널이 끊기지 않고 연속 동작하며, 답변 초안 편집까지 완성하는 E2E 흐름 검증.

## 사전 조건
- 워크벤치 페이지(`/workbench`) 접속 완료
- 좌측 "민원 목록", 중앙 "원문/요약/답변 초안/유사 민원 검색", 우측 패널이 표시됨
- 모의 데이터(mockAssignedCases) 10건 이상 로드됨

## 시연 동선 (단계별)

### 1단계: 민원 선택 → 요약 확인
**액션:**
- 좌측 민원 목록에서 임의의 민원 행을 클릭 (예: 첫 번째 민원)

**예상 결과:**
- ✅ 선택된 민원 행 배경색 변경 (흰색 강조)
- ✅ URL 쿼리 파라미터 업데이트: `?case_id={selected_id}`
- ✅ 중앙 우측 상단에 "TOPIC: welfare / LEVEL: high" 라우팅 정보 표시
- ✅ 중앙 "민원 요약" 섹션에 관찰내용/문제분석/요청사항 3열 표시
- ✅ 우측 상태 표시: "상태: 미처리" 또는 선택된 민원의 현재 상태

**검증 포인트:**
- 라우팅 정보(`strategy_id`, `route_key`, `routing_trace`)가 메모리에 로드됨 (개발자 도구 → localStorage 확인 가능)
- 좌/중/우 패널이 선택 민원 컨텍스트로 동기화됨

---

### 2단계: 요약 확인 → 유사 민원 검색
**액션:**
- 중앙 "유사 민원 검색" 섹션의 검색어 입력란에 검색어 입력 (또는 비워둠)
- "유사민원검색" 버튼 클릭

**예상 결과:**
- ✅ 검색 상태: "유사 민원을 검색 중입니다..." 메시지 표시
- ✅ 약 0.65초 후 검색 완료
- ✅ 유사 민원 리스트 표시 (5건 기본):
  - 각 항목에 "유사민원 {순번}", 제목, 스니펫, 매칭 점수(%) 표시
  - 상태: "COMPLETED"
- ✅ 라우팅 정보 동기화: 검색 결과의 `strategy_id`, `route_key`, `routing_trace` 저장

**검증 포인트:**
- 중앙 패널의 필터(카테고리, 지역)가 유사 민원 검색에 반영됨
- localStorage에 라우팅 정보 저장 확인 (workbench-routing-info)
- 검색 오류 발생 시 폴백: mock 데이터 반환 + 에러 메시지 표시

---

### 3단계: 유사 민원 확인 → 상세 펼침
**액션:**
- 리스트의 첫 번째 유사 민원 항목 클릭 (또는 "▼" 아이콘)

**예상 결과:**
- ✅ 클릭한 행 배경색 변경 (밝은 배경)
- ✅ 상세 영역 펼침: 2열 레이아웃
  - 좌: "유사민원" 섹션 — 유사 민원 제목/답변 텍스트
  - 우: "타부서 메모" 섹션 — 참고부서별 메모 목록
- ✅ 각 메모 항목: 부서명, "메모 {순번}", 민원 내용, 답변 표시

**검증 포인트:**
- 유사 민원 상세 정보가 동적으로 로드되거나 mock 데이터로 표시됨
- 아코디언 토글 동작 정상 (펼침/접힘 반복 가능)

---

### 4단계: 유사 민원 → 답변 초안 생성
**액션:**
- 중앙 "답변 초안 및 비교" 섹션의 "초안" 버튼 클릭
  - 2~3단계(유사 민원 검색)를 건너뛰고 바로 눌러도 된다. 검색이 선행되지 않았으면 "초안" 버튼이 유사 민원 검색을 **자동으로 먼저 수행**한 뒤 초안을 생성한다(검색이 만들어내는 `routing_hint`가 있어야 `/qa`가 동작하기 때문).

**예상 결과:**
- ✅ 초안 생성 상태: textarea에 "초안 생성중입니다..." 메시지 표시
- ✅ 약 1.1초 후 완료 (검색을 건너뛴 경우 유사 민원 검색→생성을 연달아 수행하므로 첫 클릭은 더 걸리고, 유사 민원 목록도 함께 채워진다)
- ✅ textarea에 초안 내용 표시:
  ```
  [단일 요청 모드]
  요청: {요청 세그먼트}
  요약: {AI 요약}
  조치 항목:
  1. {조치 항목 1}
  2. {조치 항목 2}
  
  {답변 초안 텍스트}
  ```
- ✅ 우측 패널에 라우팅 정보 유지:
  - "TOPIC: welfare / LEVEL: high" 표시
  - `citations` 목록 표시 (참고 문서 링크)
  - `limitations` 표시 ("이 답변은 프론트엔드 목업 데이터로 생성되었습니다.")

**검증 포인트:**
- 초안 생성 시 검색 결과의 라우팅 정보(`routingHint`) 연계 사용
- 초안 응답의 `structuredOutput`, `answer`, `citations`, `limitations` 모두 표시
- localStorage에 마지막 초안 저장 (workbench-last-draft)
- 초안 스냅샷 저장 (workbench-draft-snapshot) — before/after 비교용

---

### 5단계: 답변 초안 편집 → 검토 완료
**액션:**
- textarea에서 초안 텍스트 편집 (예: 내용 일부 추가/수정)
- 좌측 하단 "검토중" 또는 "처리완료" 버튼 클릭

**예상 결과:**
- ✅ textarea 내용 편집 가능 (커서 입력/선택/삭제 모두 가능)
- ✅ "검토중" 클릭: 민원 상태 → "검토중"으로 변경, 좌측 리스트 상태 배지 업데이트
- ✅ "처리완료" 클릭: 
  - 민원 상태 → "처리완료"로 변경
  - 선택 자동 전환: 목록의 다음 민원으로 이동 (또는 끝에 도달 시 현재 유지)
  - 새로운 민원이 선택되면 중앙/우측 패널 초기화 (1단계부터 재시작 가능)

**검증 포인트:**
- 편집 내용이 메모리에 유지되고, localStorage 저장 안 함 (명시적 저장 없음)
- 상태 변경 시 localStorage의 `case-status-overrides` 갱신
- 초안 생성 후 복원 로직: 같은 민원 재선택 시 마지막 초안 복원됨

---

## 예상 시연 시간
총 약 5-10분 (검색/생성 대기 포함)

## 상태 UX 4종 확인 포인트

각 단계에서 다음 상태 메시지가 일관되게 표시되는지 확인:

| 단계 | 상태 | 메시지 |
|------|------|--------|
| 유사 민원 검색 | loading | "유사 민원을 검색 중입니다..." |
| 유사 민원 검색 | success | (리스트 표시) |
| 유사 민원 검색 | error | "검색 중 오류가 발생했습니다. 필터를 다시 확인하고 재시도해주세요." |
| 유사 민원 검색 | empty | "유사 민원 결과가 표시됩니다." |
| 초안 생성 | loading | "초안 생성중입니다..." |
| 초안 생성 | success | (초안 텍스트 표시) |
| 초안 생성 | error | "초안 생성 중 오류가 발생했습니다. 다시 시도해주세요." |
| 초안 생성 | empty | (빈 입력창 · placeholder "여기에 내용을 입력하거나 AI가 생성한 초안을 편집하세요...") |

- [ ] **단계 1-5 모두 순차 실행 완료**

  - [ ] 좌측: 선택된 행 강조 + 상태 배지 업데이트
  - [ ] 중앙: 선택 민원의 요약/검색/초안 표시
  - [ ] 우측: 라우팅 정보(`TOPIC`/`LEVEL`) 유지 표시
- [ ] **라우팅 정보 상단 유지** (중앙 "민원 요약" 섹션 상단 "TOPIC: {type} / LEVEL: {level}" 표시)
- [ ] **우측 패널 편집 가능 여부 확인**
  - [ ] textarea에서 초안 수정 가능
  - [ ] `citations` 목록 표시
  - [ ] `limitations` 텍스트 표시
- [ ] **상태 UX 4종 메시지 일관성**
  - [ ] loading/success/error/empty 상태별 메시지 표시 (위 표 참조)
- [ ] **API 실패/복구 동작 (선택)**
  - [ ] 네트워크 오류 시 mock 데이터 자동 반환 + 에러 메시지

- [ ] **초안 스냅샷/영속화 확인 (선택)**
  - [ ] 초안 생성 후 localStorage 저장 확인 (개발자 도구 → Application → localStorage)
  - [ ] 같은 민원 재선택 시 마지막 초안 복원됨

## 트러블슈팅

| 증상 | 원인 | 해결 방법 |
|------|------|----------|
| 민원 선택 후 요약 미표시 | caseContext 빌드 실패 | 브라우저 콘솔 에러 확인, 목 데이터 적재 여부 확인 |
| 유사 민원 검색 결과 없음 | 필터 조건 과도 | 필터 "전체"로 리셋 후 재검색 |
| "초안 생성 중 오류가 발생했습니다" 표시 | "초안" 클릭 시 자동 선행되는 유사 민원 검색 또는 `/qa` 호출 실패 (백엔드/Ollama 미기동, `routing_hint` 누락 등) | 백엔드(`/api/v1/search`·`/api/v1/qa`)와 Ollama 기동 상태 확인 후 재시도 |
| 초안 텍스트 편집 안 됨 | textarea readonly 상태 | 브라우저 F12 → HTML 확인, readonly 속성 제거 여부 확인 |
| localStorage 용량 초과 | 상태 저장 데이터 과다 | 브라우저 개발자 도구 → Application → Clear Site Data |
